## ES6它属于js(ES6, BOM, DOM)


## let变量声明
1. 不能重复声明
```js
let a = 100;
let a = 200;     // 报错
```
2. 块级作用域
```js
{
    let gi = "Lisa";
}
console.log(gi);     // 报错
```
3. 不存在变量的提升
```js
console.log(song);
var song = "hahaha";     // 找不到
let song = "hahaha";     // 报错
```
4. 不影响作用域链
```js
let school = "广科";
function fn() {
    console.log(school);     // 向上找变量
}
fn();
```


## let案例


## const常量的声明和使用
   1. 常量拥有let所有的属性
   2. 在声明的时候必须要给它赋初始值
   3. 常量一旦声明就不能再修改
   4. 我们一般把常量用于声明一个数组或者对象 同时我们可以给数组和对象添加和删除元素


## 变量的解构赋值
1. 数组的解构赋值
```js
const F4 = ["xiao", ["liu", "zhao"], "song"];
let [xiao, [liu, zhao], song] = F4;
console.log(xiao);       // "xiao"
```
2. 对象的解构赋值
```js
const yi = {
    name: "易烊千玺",
    age: 25,
    changge: function() {
        console.log("唱歌");
    }
};
let{name, age, changge} = yi;
console.log(name);       // 易烊千玺
console.log(age);        // 25
```


## 模版字符串  
ES6中引入了新的字符串声明方式`''`、 `""`、 \````
1. 在内容中可以直接出现换行符
```js
 let str = `<ul>
           <li>aaa</li>
           <li>bbb</li>
           <li>ccc</li>
          </ul>`;
```
2. 变量的拼接 使用`${变量名}`进行拼接
```js
let wo = "let";
let shuchu = `${wo}不是一个好老师`;
console.log(shuchu);     // 我不是一个好老师
```

## ES6里面的箭头函数
```js
let fn = (a, b) => {
    return a+b;
}
let result = fn(1, 2);
```
1. this是静态的它始终指向函数声明时所在的this的值
2. 箭头函数不能作为构造函数实例化对象
3. 不能使用arguments变量
4. 箭头函数的简写
   1. 当形参有且只有一个时可以省略小括号
   2. 当代码体只有一句时可以省略花括号
   ```js
   let pow = n => n*n;
   ```

## 扩展运算符
1. 数组的合并
```js
const kuaizi = ["taili", "xiaoyang"];
const fenghuang = ["zengyi", "linghua"];
// const zuixuanapple = kuaizi.concat(fenghuang);
const zuixuanapple = [...kuaizi, ...fenghuang];
```
2. 数组的克隆
```js
const sanzhihua = ["e", "g", "m"];
const sanyecao = [...sanzhihua];
console.log(sanyecao);
```
3. 讲伪数组转为真正的数组
```js
const divs = document.querySelectorAll("div");
const divarr = [...divs];
console.log(divs);
console.log(divarr);
```


## Symbol
1. 是一种数据类型 表示独一无二的  
**U**(Undefined) **S**(String, Symbol)**O**(Object) **N**(Null, Number)**B**(Bool)
```js
let s = Symbol();
let s2 = Symbol("广科");
let s3 = Symbol("广科");     // s2 !== s3

let s4 = Symbol.for("广科");
let s5 = Symbol.for("广科");     // s4 === s5
```
2. 不能与其他数据进行运算
3. 可以解决对象里面命名冲突等问题
```js
let game = {
    up: (),
    down: (),
}
let methods = {
    up: Symbol(),
    down: Symbol(),
}
game[methods.up] = function() {
    ...
}
game[methods.down] = function() {
    ...
}
let game2 = {
    name: "wangzhe",
    [Symbol("say")]: function() {
        console.log("changge");
    }
}
```
